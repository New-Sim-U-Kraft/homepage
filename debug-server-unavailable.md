[OPEN] server-unavailable

## 症状
- 用户反馈“服务器不可用”。

## 期望
- 本地站点可正常启动并通过 `http://localhost:3000/` 访问。

## 当前假设
- H1: Node 服务进程没有启动，或之前被检查命令意外终止。
- H2: 服务启动时发生运行时异常，导致端口没有成功监听。
- H3: 端口 `3000` 被其他进程占用，当前服务启动失败。
- H4: 启动命令或项目依赖状态异常，导致服务根本未进入监听流程。

## 证据记录
- `npm start` 成功启动，终端输出 `Server running at http://localhost:3000`。
- 首页访问 `GET /` 返回 `200`。
- 开发者接口访问 `GET /api/developers/menglannnn` 返回 `200`。
- `Get-NetTCPConnection -LocalPort 3000 -State Listen` 显示本地有进程在监听 `3000` 端口。

## 假设结论
- H1: 部分成立。之前服务很可能未运行，但当前已成功启动。
- H2: 被否定。当前没有观察到启动时报错退出。
- H3: 被否定。`3000` 端口已由当前服务正常监听，不是占用冲突导致失败。
- H4: 被否定。启动命令与依赖在当前环境下可正常工作。

## 暂定根因
- 更可能是此前运行中的本地服务进程被停止，导致用户访问时出现“服务器不可用”。
- 后续复现中确认另一处更直接根因：鉴权辅助函数在已返回 `401/403/428` 后，仍把 `res.json(...)` 的返回值当作成功结果继续向下传递，受保护路由再次写响应时触发 `ERR_HTTP_HEADERS_SENT`，导致 Node 进程退出。

## 下一步
- 已修复鉴权函数失败分支，现未登录保存会正常返回 `401`，不会再把服务打崩。
- 由于服务重启后内存会话会失效，需重新登录后再验证“编辑我的主页”保存。
- 若用户重新登录后仍无法保存，再继续针对登录或前端提交流程做二次取证。
