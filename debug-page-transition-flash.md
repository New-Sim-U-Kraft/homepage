[OPEN] page-transition-flash

## 症状
- 点击站内链接切换页面时，加载过场（上划入/转圈/下划出）只闪一下（约 0.1s），肉眼几乎看不见。

## 期望
- 过场至少可见 1-2 秒：从上方划入 → 圈圈绕一圈 → 停留片刻 → 从下方划出。

## 假设（可证伪）
- H1：点击没有被过场拦截（默认导航先发生/事件未进入处理器），导致只看到下一页的极短进入闪现。
- H2：过场已显示，但在同一页内被快速隐藏（hide 被异常提前调用或样式状态被别的逻辑覆盖）。
- H3：脚本或样式未更新（缓存/版本号/多页面引用不一致），实际运行的仍是旧逻辑。
- H4：页面在显示过场后发生了“立即卸载/重载”（例如 location 被重复设置或多次触发导航），导致过场没机会跑完。
- H5：浏览器进入了 reduced-motion 或某些条件分支，跳过了动画时序。

## 采集计划
- 在前端增加仅用于调试的事件上报（不改业务逻辑）：
  - 点击捕获阶段是否进入、目标链接是否被判定为可过场
  - show/spin/hide 的调用时间点、当前 class 状态
  - 导航 setTimeout 的触发与实际跳转前的时间差
  - 页面 unload / visibilitychange 的发生时序
- 使用本地 Debug Server 收集 ndjson 日志，按时间线比对。

## 验证步骤
- 打开首页 http://localhost:3000/
- 点击任意站内链接（例如“反馈/个人主页/图集”），复现“只闪一下”
- 等待我抓取日志并给出结论与最小修复
